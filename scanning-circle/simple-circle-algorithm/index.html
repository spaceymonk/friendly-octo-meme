Circle Draw Algorithm

Pros

  • Highly straightforward implementation by expressing both in Carteesen and Polar coordinates.

Cons

  • There will be gaps. You can eliminate the gaps by changing the granularity. It can be calculated automatically for angle implementation by the formula: 
    granularity = Math.atan(1 / R)
  • High cost due to numerous floating point arithmetics.
  • But another problem arises from the angle step method. Some pixels may get painted twice.

Usage

  • Move your mouse to change end point, left click to set start point of the line.
  • Select different algorithms and grid size to experiment.
